Site Report

Overview

Learning HTML and CSS

My whole web development learning journey is chronicled on this website. I investigated fundamental ideas like HTML, CSS, layout design, version control, debugging, and responsive development while building the portfolio. The project shows the abilities I've acquired and how I used them to create a neat and useful website. Studying CSS and HTML Any website's structural framework is provided by HTML (HyperText Markup Language), and its presentation and styling are managed by CSS (Cascading Style Sheets). This module taught me about: HTML elements, attributes, and tags HTML5 semantic structure CSS properties, layout strategies, and selectors Current HTML5 and CSS3 features I used these ideas to create a neat, organized, and eye-catching design for every page in my portfolio.

Learning Git and GitHub

Git is a distributed version control system that helps track and manage changes in code. GitHub is an online platform for hosting repositories and collaborating with others. I practiced essential Git commands such as: • git clone • git add • git commit • git push • git pull I also learned about branching, repository management, and the importance of commit messages. GitHub was used to store my project and maintain version history.

Design Layout

My design approach was centered around simplicity, clarity, and professionalism. To achieve this: • I used a minimal color palette for a modern look. • Transitions and hover effects were added to improve user experience. • The layout includes a structured header, multiple content sections, and well-organized typography. • Anchor links were used for smooth navigation between pages.

Navigation Bar

The navigation bar is placed at the top of all pages to ensure consistent user experience. Key features include: • A personal logo • Links to Home, Projects, Contact, and Site Report • Flexbox for horizontal alignment • Hover effects for better interactivity The navbar is clean, responsive, and easy to navigate.

Home Page

The home page introduces me and provides a quick overview of the portfolio. It includes: • A brief introduction • An image representing me or my work • A call-to-action link directing users to my projects Flexbox was used to align the text and image. Additional styling such as padding, shadows, and hover opacity effects enhance the presentation.

Validation for index.html

The homepage was checked through the W3C HTML validator, and any structural issues were corrected to ensure valid and clean markup.

My Projects

Responsive Design

To ensure the website works on all devices—including laptops, tablets, and phones—I used: • The viewport meta tag • Media queries • Flexible units (%, rem, vh) • Responsive images and layouts These techniques allow the website to adapt smoothly across various screen sizes.

Challenges

During development, I faced several challenges: • Responsive layout issues • Browser compatibility differences • Managing spacing and alignment • Attempting to implement a hamburger menu for mobile navigation Although the hamburger menu caused alignment problems and is not fully functional yet, I plan to improve it in future updates.

Conclusion

This project helped me gain a strong understanding of HTML, CSS, Git, and GitHub. By building this portfolio, I improved my coding skills, design sense, and problem-solving abilities. The website reflects my progress and serves as the foundation for my future web development journey.
